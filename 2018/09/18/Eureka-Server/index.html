Eureka Server

发表于 2018-09-18 | 分类于 spring cloud | 评论数：

主要是Server端的一些配置。

总体

Eureka Server内部内置了一个Client，通过这个Client端向着其他Server去注册。所以Client端能用的，这里基本都能用。

最小配置

这里唯一有点注意的就是zone怎么配。在官方文档里，对于zone几乎没有描述。查了一些文档之后，其实zone是分区的概念。可以用，也可以不用。

注意点，也许在写例子的时候配了localhost会出一点问题。不清楚是显示问题，会不会有业务逻辑基于这些。

  1. 如果配了Localhost。General Info-available-replicas一只为空，而是在unavailable-replicas里面。但是注册的信息会相互沟通

具体看stackoverflow-jebeaudet

  1. 要配hostname

多服务器配置

  1. 配置多个服务器，不能用yml的数组表示，必须要用逗号。感觉，配置就是把相应的Class映射到对应的yml里面。

  2. 注册是单向的。 举个例子来说，如果A和B相互注册，C如果向A注册。那么会有以下几点。
    • 注册到C的服务，会同步到A，但是注册到A的不会，同步到C，由此可一直，其实同步，是一个推得过程
    • A和B会相互为Replica。但是C不是A的replica
