command

两种不同的记录方式

CMD

  • 和RUN的区别，在于是容器启动时运行的命令
  • 会被Docker run所==覆盖==。特别是参数
  • 一个DockerFile中，==只能运行一个==

ENTRYPOINT

  • 基本和CMD功能相近
  • 不太容易被docker run命令覆盖：说句实话，这个我还真的不太明白什么意思。
  • docker run的的参数会传递给这个命令：上一点的实例？

smart example

这里，如果在启动容器的时候，外面有参数，那么就会把CMD [“-h”]给覆盖。否则，就用CMD的做为默认参数

WORKDIR

指定工作目录

ENV

环境变量

USER

制定运行的用户

VOLUME

现在的理解，在于使得两个container之间可以互相分享文件。
在DockerFile中，这个类似于宿主的功能。表示，这块区域可能被开放。
其他的需要 ==--volumes-from== 来指定其用的哪一个container的volume。而且是全部都会使用。

ADD

将构建环境下的文件和目录复制到镜像中 构建环境，就是DockerFile所在的文件夹

Volume

-v $PWD/website:/var/www/html/website 现在对于Volume还刚刚开始了解。不太了解具体的含义。这句命令就是在启动的时候创建一个卷。ranhou $PWD/website。为本地的目录。后面的为container上面的目录。 效果感觉上像是mount上去。

主机和container上的的权限是==相同的==。在主机上的权限，

命令行中的-v。建立起了一个==contianer和宿主机的桥梁== 。使得两者可以共享文件。

NetWork

网络连接的方式。我觉得是一种建立一个虚拟的局域网。然后在这个局域网中，用container名字进行通信 ## Basic - 书中，所说的是其会在/etc/hosts内写入。但是我发现已经不是 - 基本命令格式，很简单，就是docker network create/connect/disconnect - 每个container的名字，就是其内network里的名字。run的时候用参数–net

老版本（1.9）之前

用–link，看懂就好了。 –link container_name:alias_name

daemon.json

位置

/etc/docker/daemon.json ### 参数 #### registry-mirrors 就是镜像的地址 #### insecure-registries 指定访问mirror是用http的。貌似docker默认都是走https的。如果不放在这里，都会默认开https去访问。返回一个客户端不匹配的错
server gave HTTP response to HTTPS client

装ping

一个宿主和container共享文件的思路。

在container中设置一个Volumne。然后新建一个运行后删除的Contianer（run 加上 -rm）。这个container，启动的时候用-v来做一个和宿主建立link。这样就能把文件给弄下来了。
主要用途
- logger：貌似还有一个专门弄log的==logstash==

其实还可以同步/etc/timezone,但是实际操作中，这个要访问权限，貌似不是默认开的
