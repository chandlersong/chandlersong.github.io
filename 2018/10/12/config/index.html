bootstrap.yml 在客户端的配置，都要写在这个文件中，因为要在启动前确定

貌似要在这里手动加上enabled: true才能工作。这点让我感觉很奇怪。因为官方文档中没有提。所以我也不太清楚到底是我的问题

prifile

两种定义profile的方式。一种是系统自己的

另一种是定义在config里面

孰优孰劣很难说，前者比较灵活一定，你的profile不用跟着统一，后者的配置量减少。

如果要默认，配置的文件可以加上default

对应关系

{application}, which maps to spring.application.name on the client side. {profile}, which maps to spring.profiles.active on the client (comma-separated list). {label}, which is a server side feature labelling a “versioned” set of config files.
