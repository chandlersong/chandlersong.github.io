GO-自学入门4

发表于 2019-09-23 | 分类于 Go | 评论数：

Go 自学笔记。主要还是在读《Go语言实战》的第五章的笔记主要是关于类型的

自我感想

  1. Go的OO比较偏向于Js那种的protoype。就是有一个原型，然后你慢慢的给他加入方法以使其产生质变。从而产生新对象
  2. 方法和数据和分离的。方法是方法，数据是数据。其实你没有办法顶一个struct的时候，能够决定他到底是什么东西。
  3. 定义方法很繁琐。要考虑这个方法的对象是传值还是传址。
  4. 内嵌类型感觉像是继承。但是感觉在GO中，继承其实用处不大。
  5. itable感觉像是GO定义方法的基本，但是估计用到分析的概率不大。

。 一种语言有其内部的逻辑和理念，不要用自己习惯的语言去套。

笔记

  • 定义的方式就是type xxx struct
    • 属性和方法是分离的。这个觉得很蛋疼。因为java的习惯了，这个真的有点不习惯。
      • 以前听说过OO有两种方式，一种是C++的，而另一种，类似于Js那样添加方法。估计GO比较像是后一种。
    • 方法接受者，要分清楚是传值，还是传址
      • 传值的话，会复制一份，换句话说，方法里面的修改，其他地方不可读
      • 传址的话，方法里面的变换能够影响到外面
      • 觉得这个还是有点蛋疼的，方法里面的self，居然要考虑会不会复制，这个考虑的问题有点多。
    • 外部调用方法的，传址和传值会自动转换。
    • 所有的类型要考虑一个问题，就是传址还是传值。
      • 内置类型：数值，字符串和布尔。传递副本
      • 引用类型：切片，映射，通道，接口和函数
        • 这时候，变量被称为标头，我的理解是类似指针
      • 结构类型： 自定义类型
      • 看了N多。简单的逻辑，就是区分原始和非原始
        • 这个区别的核心，是这个值是否一个值，那么就是原始的
        • 是一个对象，那就是非原始的。
        • 我觉得这里把问题给搞复杂了。就是把对象和值搞成了一个概念。我觉得他们是两个维度的不同表达。具体可以看看java中的lombok中的@Data和@Value
  • interface: 在所有面向对象的语言里面，都明白interface的作用。而在GO里面，interface更加的纯粹，因为Go要求一个interface只有一个方法。我的第一门语言是java，java的interface不如Go纯粹。往往是归集。而不是一个接口
    • 因为Go的类型模型是鸭子模型。所以inferace只是一种定义。严格的来说，实现者，可能完全不知道他实现了某个接口。这个其实有点恶心的
    • 内存里面，处理interface的时候，图5-2比较形象的说明了内存里面的对象。
      • 两个指针，一个是itable的指针，指向有多少方法
      • 一个是对象的指针
    • itable就是一个方法集，用来匹配，这里其实我看的不是很深。简单的说从实现角度来说
      • 如果接收者是一个值，那么传入的可以是一个值或者指针
      • 接收者是一个指针的话， 传入的只能事指针
      • 为什么这样，说是值无法马上获得地址，不过这个不太明白。
  • 嵌入类型embedding感觉就像是继承
    • 两种调用方式是等效的。换句话说，如果子类要调用父类的方法，其实很简单就是用第二种调用
    • 多重继承要报错。
  • 公开和私有
    • 所有都是package一级的
    • 小写开头是私有，大写开头是外部可以访问
    • 工厂方法，建议用New
