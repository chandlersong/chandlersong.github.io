Go-自学入门2

发表于 2019-09-14 | 分类于 Go | 评论数：

Go 自学笔记。主要还是在读《Go语言实战》的第三章的笔记

个人感想

  • goruntine像是整个GO中的核心。不过我觉得有点像是语法糖。看下来，其本质，还是把一段逻辑封装起来，然后交给语言去运行。
    • 因为没有看过goruntine，有没有其他功能或者特性，但是应为是语言级别的。性能可能比其他语言要高一点
  • 整体语法比较像C和C++，因为有指针
  • OO这块，并不是走C++的模式。而是组合模式
  • 错误处理很ugly。
  • 有些语法过细
    • 比如说大写可以外部访问，小写开头不行
    • 入口只能放在main包下。

第一天

  • goroutine: 这东西感觉在Go里面，就像是线程一样的东西。但是不同于线程。因为文中说了，其可以一个线程，run几个goroutine
    • 感觉这个像是集装箱一样的性质。把其他编程中，线程中，封装到一起，进行处理。然后由后台统一调度。
  • Channel：不同goroutine之间相互信息传递。保证了一个时间，只有一个goroutine修改一个Channel
    • 看书的时候，说到不保证跨goroutine的数据访问保护机制。
      • 副本是安全的。也就是传值
      • 指针是不安全的。也就是传地址
  • OO相关
    • 采用组合（composition），而不是传统的继承
    • 支持鸭子类型，类型校验不是很严格。
  • 自动内存管理
  • playground

第二天

  • 看了第二章开头那张图之后，最后还是证实了我以前的想法。goroutine其实是一个逻辑的业务模块。
  • 程序关键点
    • main:程序入口，参数为空
      • 还要在main包里这点还是挺奇怪的。
    • init:初始化
  • 语法特点
    • import之前有_表示载入的时候，init方法都会被调用。
    • 回到GOROOT和GOPATH下面，寻找需要导入的包
    • 小写字母开头，是内部变量。外部不能范根
    • 大写字母开头，是外部变量，外部能够直接访问
    • feeds, err := RetrieveFeeds()
      • err标识返回的错误
      • := 标识申明变量并且赋值
      • _, err := RetrieveFeeds() 标识忽略这个值。
    • main函数返回，程序终止
      • 会关闭所有之前启动的goroutine
    • go func(xxx,yyy) 启动匿名函数
    • *Feed 表示是指针
    • const 常量
    • 接口只有一个方法，这个主要还是因为Go的类型是组合的。
      • 接口，最好以er结尾。这个是命名规则
    • func (m defaultmatcher) search。方法和类型绑定在一起。
      • 方法是search
      • (m defaultmatcher) 是一个类型
