发表于 2019-09-17 | 分类于 Go | 评论数：

Go 自学笔记。主要还是在读《Go语言实战》的第四章的笔记 主要是数组，切片和map

个人感想

array和map基本都是基本操作，每个语言都差不多的东西。

切片(slice)觉得是一个不错的炫技的东西。但是不能把他当成java的List来用。主要是以下几点

  • slice的底层是array。但不是List完全封装的。感觉上，从设计理念上来说，他更像是可变的array，而不是List
  • 如果要对slice进行append操作。那么要考虑capacity这个是否会超。感觉这个是挺蛋疼的操作。因为一旦超了，slice就会生成新的底层数组复制，对底层数组的影响就不一致
    • 由于这一条，感觉slice不太适合做为参数的返回值，除非是说明只能事只读。
  • 官方的几个骚操作
    • 如果不触发底层数组的复制，性能会很快。这样写很骚。否则，估计会很慢。
    • 觉得这类如果做为第三方类库可能更好，更安全一点。
    • 其实可以分装成方法。

看书笔记

  • 数组，基本算是常规操作，没有什么特别
    • 初始化的时候不定长的办法[...]int{1, 2, 3, 4, 5},就是[...]。
    • 复制的时候，要一样大小。否则会编译不过。
    • 作为方法的时候，最好是传址，因为Go里面，都是复制。如果复制的话，要整个的数组
  • 指针: 里面有很详细的说明，其实对于像用java这类的我来说，有点不习惯。不过记忆一下就好，就是在传递大对象的时候，就要考虑这个问题。
  • String format语法
  • map: 创建方式有两种。make(map[string]int) 和 map[string]int{"red": 1}
    • 方法参数传的是值，不是址。这个感觉有点神奇
    • 删除操作很神奇delete(dict2, "red")
    • 判断存不存在最好以一下方法，value, exists := dict1["blue"]。尽管来说，exists不存在也提供了方法，但是万一默认为空呢？

切片(Slice)

  • 两个创建语法make([]int, 5)和[] int{1, 2, 3, 4, 5}
  • 切片是数组或者队列这类的数据结构，而是一种对数组的操作方式
  • 三个组件
    • 指向数组头的指针
    • 标识可以访问的长度
    • 数组的容量，不能小于长度
  • 空切片 var emptySlice [] int
  • 官方说明
    • Slice的几个用法

append

说句真的，觉得这个方法好恶心。有点行为不一样。因为共享内存， 当用了append之后，

  • 长度小于等于容量的时候，则是会修改原底层数组。
  • 长度大于容量的时候，则会创建一个新的底层数组。
    • 增长的策略是翻倍。
