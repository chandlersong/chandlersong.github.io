爬坑

  • Got permission denied while trying to connect to the Docker daemon socket

网上找了很多的方式，基本都是给权限。但是发觉那个默认的jenkinsci/blueocean居然没有命令usermod这个命令。 其他方案

发觉还可以用--group-add。在容器启动的时候加入用户组。每个可能不一样。可以去容器自己看。而且容器里的组和宿主机上面的组不一样。就像是我的例子中，宿主机的group是docker，而容器里面是ping

  • 环境变量不能作用于agent
    • stackoverflow上讨论：提到了其实script有两种语法declarative和script。jira上的是script语法
    • 官方jenkins上面提供了一个解决方法:
  • volumn的permissions denied：我的需求是想要做一个volume。然后把build好的文件全都放进去。但是发觉最后操作的时候发觉有点在copy的时候，一直permissions denied，看了一下，目录的是root的
    • -v rumor_mill_distro:/opt/rumor_mill:这个是参数
    • 解决方案，就是强制给这个文件夹777权限。有点暴力了。
    • 类似的问题分析，是SELINUX相关：和我不同的地方在于，我的jenkins和build的都是contianer。而我的宿主机本身就是ubuntu
    • stackoverflow上面讨论：基本思路就是启动的时候指定用户。但是我发觉这个不解决本质问题，因为我的问题是，相关目录都是root
    • 另一个思路，没试验成功:--privileged:例子中是--privileged=true,有点不一样
  • 如果以docker启动，他都会把volume通过把workspace给mount到新的contianer上。
