《zookeeper_distributed_process_coordination》 第一章笔记

基本功能

Synchronization primitives

同步原始信息。其中包括 - leader election - locks - Configuration metadata

zookeeper提供的功能

  • Strong consistency, ordering and durability guarantees
  • the ability to implement typical synchronization primitives
  • A simpler way to deal with many aspects of concurrency that often lead to incorrect behavior in real distributed systems.

zookeeper不做的事情

  • is not for bulk storage
    • spartes application data from control and coordination,that’s for zookeeper
  • zookeeper does’t implement the tasks. but offer the tools to do the these tasks for the distributed application.

distributed systems with zookeeper

advatage

  • commponent to be designed and implemented independently
  • enable system architect to reason more easily about corrdination

faceing problem

  • Message delays
  • Processer speed
  • Clock drift

it’s very hard in practice to tell if a process has crashed or if any of these factors is introducing some arbitray delap

Examples of Master-Worker application

three key problem - master crashes: need a backup master - split brain - worker crashes - detect the worker crashes
- Communication failures - Exactly-Once and At-Most-Once Senmantics - synchronization primitives like locks - zk use ephemeral node" - zk will periodically check node is alive

requirment of master-worker architecture - Master election - Crash detection - Group membership management - meta management
