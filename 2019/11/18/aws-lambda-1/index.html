第四天

  • aws region list
  • 关于lambda
    • 其实lambda是和每个账户相关的。
    • 默认调用，类似于cli形式，不是restful的形式。也不是网络的api。只是调用一个命令。这点和一开始想的完全不同。
      • 建议还是用s3-proxy：来做远程调用。相当于多了一个东西而已。
  • 创建lambda的时候，需要调用aws configure配置一下用户和default region。这些信息。可以通过上面来获得。
  • 现在这些基本都是命令行相关的东西。或者说lambda其实就是一个放在网上的命令行。然后你通过某个账户来调用而已。
  • 要用restful的接口，需要用gateapi来弄
  • aws这个工具打help，其实还是挺简单的。就是不用打--
  • 少有的第三方介绍：看大不懂。
  • 里面有个第三方的工具，npm install -g serverless 挺好用的
