《Nodjs实战》 第二章读书笔记

个人总结

相对于以前的javascript，nodjs给我最大的感觉，其实是模块化和标准化。记得最早的时候，想jQuery这类，是比较方便的工具，但是需要自己去组织js文件，和载入顺序。也就很麻烦。
而在Nodjs中，则是有了以下三点最大的改变。

  • export机制，是的每一个文件都是一个模块。而你在写自己模块的时候，则是会考虑哪些需要输出，哪些不需要。这点其实很重要。
  • npm的安装机制。解决了依赖管理问题。这个对实际开发，帮助其实挺大的。
  • require的载入机制。这个其实和export相辅相成。唯一有点遗憾的，就是这个不是异步。

而在语言层面。最大的特点，就是发自骨髓的异步。比如callback,event，async这些，都是使得写异步很方便。

其实作为程序员，对于异步往往是又爱又恨，谁都知道异步是好东西。真要写。最后发现的问题，还是在于麻烦二字。但是这种深入骨髓的异步，使得程序写起来，得心应手很多。

阅读感想

第一天

  1. 现在貌似不支持ES6。还在试验阶段。
    • 官方说法:大致是分为三类
    • 网上找的关于强行enable的:主要还是用Babel这个插件。最后发现其实本质是把es6转换成5
  2. const hello = require('./libs/helloworld'); 加载自己的语法。关键点，还是前面的./libs不能直接写那个/libs
  3. 总人口为index.js
    • package.json文件中配置main这一项
  4. exports其实是module.exports的一个简化。
    • exports= sayHello;: 这个是不对的。因为这样就是覆盖了原来的值了。
  5. require的加载顺序，具体看书中的图吧。听清楚的。

第二天

  1. http.createServer可能被掉两次：因为还有除了本url外，还可能是其他的url。比如/favicon.ico
  2. pub/sub模式
    • 带参数的话，就是在后面发射的地方加上去
  1. String format
    • template, 记住，特殊符号`.tab上面那个，而不是双引号那个。
    • stackoverflow上面介绍的几种方式
      • console.log('count: %d', count);

第三·天

  1. UT
    • 很奇怪，要装一个叫做Mocha的东西。官网链接。简单的来说，就是用Mocha来作为原生支持。
    • 还有一个叫做Chi，两者的简单介绍
    • 用Mocha的话，webStrom调用Mocha，类似于run了一句下面的命令.Mocha ut.js
      • describe和it，会有报错,追求极致美观的话。可以去掉。
  2. 闭包官方教程
  3. customizer event emittor：包装一下他的emitor方法，然后其他的都是和原始的一样。
  4. 专业术语：在nodjs中的异步执行顺序，叫做流程控制,常用库为以下几个。
    • async
    • Step
    • Seq
  5. async
    • 不是默认安装包，需要手动安装。npm install async
  6. 默认都是并行的。
